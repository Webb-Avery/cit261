Animations!

Designing CSS3 Animations

You can make Animations in the same way we created Transitions and transforms but with animations we can do more then one thing at the same time.

Defining CSS3 Animations

There are many different properties that we can change for animations. The actual animation will always be declared in a @keyframes section. This is what will happen to the element.
In the element that we are changing we can define any of these properties.
- animation-delay: This will tell the element how long to wait before starting the animation.
- animaiton-duration: We can play the animation forwards, backwards or in alternate cycles
- animation-duration: How long will the animaiton last.
- animation-name: This is where you tell the element the name of the @keyframes you will be referencing. - animation-iteration-count: How many times the animation will be played.

Triggering CSS3 Animations

We can have animations start when the page first loads or when can have delays on the Animation or just like transitions and transformations we can use any of the HTML Event Attributes to trigger an Transition. If we want a trasition to happen when the page is loaded for example we can trigger on the onload event. Probably the most common places to add transitions is on Mouse Events.
Here are some mouse events we could trigger our transition on:
- onclick
- ondblclick
- onmouseover
- onmousewheel
- ondrag
- onmousemove
